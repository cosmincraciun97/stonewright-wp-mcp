## Purpose
Experimental features in Elementor allow users to access and test new, in-development functionality before official release. This enables early adoption of upcoming tools and capabilities while providing feedback to the development team.

## Use this when
- You want to preview unreleased features ahead of general availability
- Testing new functionality in a controlled environment before production use
- Providing feedback on beta features to the Elementor team
- Exploring advanced capabilities not yet in standard releases
- Evaluating whether experimental tools meet your project needs

## Settings highlights
- Access experimental features through Editor Settings
- Toggle individual features on/off independently
- Features marked as experimental may change or be discontinued
- Some experimental tools may have limited functionality
- Settings persist across editor sessions once activated
- Experimental status clearly marked in feature documentation
- Can disable features anytime if they cause issues
- Advanced users can access beta versions through Developer Edition

## Limits / gotchas
- Experimental features may be unstable or incomplete
- Breaking changes possible between updates without notice
- Limited support availability for experimental functionality
- Features may be removed entirely before reaching stable release
- Performance impact possible when testing multiple experimental tools simultaneously
