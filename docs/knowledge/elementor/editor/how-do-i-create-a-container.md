## Purpose
Flexbox containers provide a responsive layout system that automatically arranges child elements in a flexible, flowing manner. They adapt to different screen sizes and content volumes, making them ideal for building modern, adaptable page structures without rigid column constraints.

## Use this when
- Building responsive layouts that need to reflow across device sizes
- Creating flexible navigation menus, headers, or footers
- Arranging elements that should distribute space intelligently
- Designing components requiring alignment control (vertical/horizontal)
- Building layouts where element order matters for different breakpoints

## Settings highlights
- Layout direction (row/column) to control element flow orientation
- Justify content options for horizontal spacing distribution
- Align items controls for vertical element positioning
- Flex wrap settings to determine wrapping behavior
- Gap/spacing controls between container children
- Alignment properties (start, center, end, space-between, space-around)
- Responsive controls adjustable per device breakpoint
- Size and behavior settings for container dimensions
- Advanced tab options for custom flex properties

## Limits / gotchas
- Flexbox works best for single-direction layouts; use Grid containers for 2D grid structures
- Deeply nested flexbox containers can impact performance on complex pages
- Element reordering via flex properties doesn't affect HTML source order for accessibility
