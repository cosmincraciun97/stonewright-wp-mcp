## Purpose
Anchor links enable navigation to specific sections within a page by creating clickable links that jump to designated target locations, improving user experience for long-form content and multi-section layouts.

## Use this when
- Building single-page websites with multiple sections requiring internal navigation
- Creating table of contents or jump-to-section functionality
- Designing long-form content pages where users need quick access to specific areas
- Implementing smooth scrolling navigation menus linking to page sections

## Settings highlights
- **CSS ID field**: Assign unique identifiers to elements serving as anchor targets
- **Advanced tab**: Located in element settings where anchor configuration occurs
- **Link URL configuration**: Use hash syntax (#) to reference target section IDs
- **Menu Anchor widget**: Dedicated widget for creating anchor points throughout pages
- **Display conditions**: Control anchor visibility based on user roles or device types
- **Smooth scroll effects**: Enable animated scrolling between anchored sections

## Limits / gotchas
- CSS IDs must be unique per page to prevent navigation conflicts
- Special characters in IDs can cause linking failures; stick to alphanumeric characters and hyphens
- Anchor links require properly configured target elements; misconfigured IDs result in broken navigation
