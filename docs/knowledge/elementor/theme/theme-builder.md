## Purpose
The Theme Builder enables you to design and customize global site templates (headers, footers, archives, single pages) that apply across your WordPress site. You control layout, styling, and display rules without editing individual pages, establishing a consistent visual and structural foundation for your entire site.

## Use this when
- Creating a unified header/footer design that appears on all pages
- Building custom archive templates for blog posts, categories, or product listings
- Designing single post or product page layouts with reusable structures
- Setting conditional display rules (show different headers on specific pages)
- Applying consistent styling through theme styles and global colors

## Settings highlights
- **Site parts**: Header, footer, archive, single post/product templates
- **Display conditions**: Show/hide templates based on post type, taxonomy, specific pages
- **Sticky headers**: Create persistent, scroll-aware navigation elements
- **Archive templates**: Customize blog, category, and product archive layouts
- **Single templates**: Design post and product page structures with dynamic tags
- **Theme styles**: Establish global typography, colors, and spacing rules
- **Responsive design**: Adjust layouts per device breakpoint
- **Template library**: Access pre-built kits and import/export templates
- **Conditional rules**: Apply templates only to matching content criteria

## Limits / gotchas
- Theme Builder requires Elementor Pro for most advanced features
- Display conditions must be carefully configured — conflicting rules may prevent template display
- Changing domain names can disrupt template functionality; plan migrations carefully
