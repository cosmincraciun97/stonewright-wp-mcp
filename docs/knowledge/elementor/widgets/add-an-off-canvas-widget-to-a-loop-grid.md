## Purpose
Add an Off Canvas widget to a Loop Grid to create dynamic, space-efficient content displays that reveal additional information in a sliding panel triggered by user interaction within repeating grid layouts.

## Use this when
- You need to display large amounts of product or post data without cluttering the main grid view
- Building e-commerce archives where users can preview items in a side panel
- Creating filterable content collections with detailed previews
- You want to maintain grid layouts while offering drill-down functionality
- Designing responsive layouts where off-canvas panels adapt to device size

## Settings highlights
- **Off Canvas trigger**: Configure which grid elements activate the panel (typically buttons or cards)
- **Animation direction**: Choose slide direction (left, right, top, bottom) for panel entrance
- **Width/height controls**: Set panel dimensions relative to viewport
- **Overlay behavior**: Toggle background overlay and configure its opacity/color
- **Close triggers**: Define how users dismiss the panel (close button, overlay click, ESC key)
- **Content binding**: Link loop dynamic data to populate off-canvas content
- **Z-index settings**: Control stacking order relative to other page elements
- **Responsive adjustments**: Modify panel behavior and positioning per device breakpoint

## Limits / gotchas
- Off-canvas panels may require custom CSS for complex nested content layouts
- Performance impacts when grid contains many items with heavy off-canvas content
- Mobile usability requires careful trigger placement to avoid accidental activation
