## Purpose
The Accordion widget organizes content into collapsible sections that users can expand and collapse individually. Each section contains a title and hidden content that appears when the user interacts with it, making it ideal for FAQs, feature lists, and space-efficient content presentation.

## Use this when
- Creating FAQ sections where users need to browse multiple questions without seeing all answers simultaneously
- Building feature comparison lists or product specification breakdowns
- Organizing lengthy documentation or terms of service into digestible chunks
- Designing content-heavy pages where vertical space is limited
- Implementing progressive disclosure patterns that reduce cognitive load

## Settings highlights
- **Items** – Add, remove, and reorder accordion sections with title and content fields
- **Title HTML Tag** – Set semantic heading level (H1-H6) for each accordion title
- **Trigger** – Specify click behavior (on title click, icon click, or both)
- **Icon** – Choose which icon displays and control its position (left/right)
- **Toggle behavior** – Allow multiple sections open simultaneously or restrict to one active section
- **Animation** – Control expand/collapse speed and easing effects
- **Tab styling** – Customize background colors, borders, padding for active and inactive states
- **Nested elements** – Support placing containers and other widgets inside accordion items
- **Active section** – Define which tab opens by default on page load

## Limits / gotchas
- Mobile interactions depend on click target size; ensure titles and icons have adequate touch-friendly dimensions
- Nested complex layouts within accordion items can impact performance if many sections are on one page
- Accordion height animations may behave unexpectedly with dynamic content of varying lengths
