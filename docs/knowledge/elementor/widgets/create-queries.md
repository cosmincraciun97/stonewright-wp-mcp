## Purpose
Queries enable you to dynamically pull and display content from your WordPress database within Elementor widgets. This feature allows you to filter posts, products, and custom post types based on specific criteria, creating dynamic loops and archives that automatically update as your content changes.

## Use this when
- Building archive pages, blog layouts, or product catalogs that display multiple items
- Creating filtered content displays that change based on taxonomy, meta fields, or post attributes
- Designing loop grids or carousels that need to pull specific content subsets
- Setting up dynamic category or tag pages with customizable filtering
- Displaying related posts or products based on query conditions

## Settings highlights
- Post type selection — choose which content types to query (posts, pages, products, custom types)
- Taxonomy filters — filter by categories, tags, or custom taxonomies with include/exclude options
- Meta queries — target specific custom field values and conditions
- Ordering controls — sort by date, title, popularity, or custom meta values
- Pagination settings — configure how results display across multiple pages
- Post count limits — set maximum items displayed per page or in total
- Advanced rules — combine multiple conditions with AND/OR logic for complex filtering

## Limits / gotchas
- Complex queries with multiple nested conditions may impact site performance
- Custom post types require proper registration before appearing in query options
- Meta queries require exact field names and value matching to function correctly
