## Purpose
The Flip Box widget creates interactive card elements that reveal different content when users hover over or interact with them. One side displays initial content (typically an image and title), while the reverse side shows additional information, making it useful for showcasing team members, services, or product features in an engaging, space-efficient manner.

## Use this when
- Displaying team member profiles with photos on front and bios on back
- Presenting service offerings with icons/images and detailed descriptions
- Creating portfolio galleries where hover reveals project details
- Building interactive feature lists for products or offerings
- Designing testimonial cards that flip to show full customer feedback

## Settings highlights
- Front and back content areas with independent styling options
- Flip animation direction (horizontal, vertical, fade effects)
- Image and text customization for both sides
- Link configuration to make cards clickable
- Border, shadow, and background color controls
- Hover trigger settings for desktop vs. mobile responsiveness
- Icon and button placement options
- Text alignment and spacing adjustments

## Limits / gotchas
- Mobile tap behavior may differ from desktop hover, requiring explicit interaction design
- Performance can degrade with multiple flip boxes on single page due to animation overhead
- Content on reverse side must be carefully sized to avoid overflow or text clipping
- Accessibility considerations needed — screen readers may not properly announce flipped content without additional ARIA labels
