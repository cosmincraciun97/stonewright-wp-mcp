## Purpose
Global widgets allow you to create reusable widget instances that can be placed across multiple pages and posts. When you update a global widget, all instances update simultaneously, ensuring consistency and reducing manual editing work across your site.

## Use this when
- You need the same widget or content block appearing on multiple pages
- You want to maintain consistent messaging across your site
- You need to make updates in one place that reflect everywhere
- You're building headers, footers, or sidebar content
- You want to streamline your workflow for frequently-used components

## Settings highlights
- Save any existing widget as a global widget via context menu
- Access global widgets from the widgets panel for easy insertion
- Edit global widget definition to update all instances site-wide
- Create new global widgets from scratch in the library
- Convert global widgets back to regular widgets when needed
- Organize and manage all global widgets from a centralized location
- Link multiple page instances to a single global widget definition

## Limits / gotchas
- Global widget updates apply universally — be cautious with broad changes
- Some dynamic content may behave differently in global context
- Deleting a global widget definition removes it from all pages using it
- Local overrides aren't possible; all instances must match the global definition
