## Purpose
Add entrance animations to page elements so they animate into view when visitors land on or scroll to your page. These effects trigger automatically when elements become visible, enhancing visual interest and drawing attention to key content.

## Use this when
- You want elements to fade, slide, or bounce into view as the page loads
- Certain sections should grab attention when scrolled into viewport
- You're creating landing pages or hero sections that need visual impact
- You want to improve user engagement through motion effects
- Designing call-to-action sections that benefit from animated emphasis

## Settings highlights
- **Animation type**: Choose from effects like fade, slide, zoom, bounce, flip, and rotate
- **Duration**: Control how long the animation plays (typically 300ms-2000ms)
- **Delay**: Set when animation begins after element appears (useful for staggered effects)
- **Easing**: Select animation curve (ease-in, ease-out, linear, etc.)
- **Trigger point**: Define scroll position where animation activates
- **Repeat**: Configure if animation plays once or loops
- **Direction**: Specify slide direction (left, right, up, down for applicable effects)

## Limits / gotchas
- Animations may impact page performance if overused on many elements simultaneously
- Mobile devices may render animations differently; test responsively across devices
- Animations don't work on all widget types; verify compatibility before applying
- Excessive animations can create poor user experience and distract from content rather than enhance it
