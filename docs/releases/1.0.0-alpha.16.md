# Stonewright 1.0.0-alpha.16

Release date: 2026-06-15

This alpha improves first-run setup and public onboarding. The main path is now
the WordPress admin Configuration page: enable Stonewright, generate an
Application Password, copy one setup note, and add Playwright MCP for visual QA
when needed.

## Highlights

- Adds a branded Configuration flow with direct Application Password generation,
  one copyable MCP setup note, per-client JSON snippets, and real-world prompt
  examples.
- Adds revoke actions for existing Application Passwords so site owners can
  manage credentials without leaving Stonewright.
- Simplifies the optional companion HTTP bridge into an advanced-only section
  with generated bridge tokens and hidden developer launch values.
- Adds MCP-facing privacy guidance so site-specific memory, site skills, and
  custom instructions stay local to the current WordPress install.
- Updates public docs with clearer `npx @stonewright/companion@latest` setup
  and examples for Figma to Elementor V3, ACF, CPT UI, WooCommerce, and
  Gutenberg/FSE work.

## Release Assets

- `stonewright-1.0.0-alpha.16.zip`: installable WordPress plugin package with
  production Composer dependencies.
- `stonewright-companion-1.0.0-alpha.16.tgz`: Node companion package for
  WP-CLI, batch WP-CLI, health checks, optional MCP proxy, setup profiles, and
  Stonewright MCP prompts.
- `SHA256SUMS.txt`: checksums for release assets.
