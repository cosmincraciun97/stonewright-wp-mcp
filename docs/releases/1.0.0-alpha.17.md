# Stonewright 1.0.0-alpha.17

Release date: 2026-06-16

This alpha hardens first-use MCP behavior. Stonewright now tells agents to
verify the live MCP tool list before WordPress work and to stop with a clear
reload/config message when the server is not loaded.

## Highlights

- Strengthens the Configuration page setup note with a clear after-reload
  checklist: verify `stonewright-context-bootstrap`, call bootstrap/preflight,
  and do not replace live MCP tools with local agent skills or repository files.
- Adds permanent default MCP instructions that stop agents from starting a task
  by only announcing named skills.
- Adds companion setup-profile notes for the same live-tool verification flow.
- Clarifies that `/wp-json/stonewright/v1/abilities/run` shell calls are not an
  MCP workaround when the Stonewright server is missing from the client.
- Updates README, installation, onboarding, and admin connection docs with the
  same MCP startup contract for open-source users.

## Release Assets

- `stonewright-1.0.0-alpha.17.zip`: installable WordPress plugin package with
  production Composer dependencies.
- `stonewright-companion-1.0.0-alpha.17.tgz`: Node companion package for
  WP-CLI, batch WP-CLI, health checks, optional MCP proxy, setup profiles, and
  Stonewright MCP prompts.
- `SHA256SUMS.txt`: checksums for release assets.
