# Stonewright 1.0.0-alpha.65

Release date: 2026-07-14

> Post-release audit: the release did not yet contain a real
> `stonewright/task-start`; its published 603/646 token figures measured a
> different bootstrap payload. The 216 cases audited curriculum structure, not
> live implementation behavior. See the
> [roadmap audit](1.0.0-alpha.65-roadmap-audit.md) for corrections and remaining
> controlled-site gates.

This alpha rebuilds Stonewright around compact discovery, exact live schemas,
native-first implementation, and editor-resident Visual adapters.

## Highlights

- Keeps the default MCP surface at 20 tools and strict-minimal at 12 while the
  full internal ability catalogue remains available for diagnostics.
- Adds a single `stonewright-workspace-request` Visual gateway with nested
  Elementor V3, Elementor V4 Atomic, Gutenberg, backend, confirmation, and
  transactional batch tools.
- Discovers all installed Elementor V3 widgets and V4 Atomic element schemas
  from the live runtime; unknown settings and silent V3/V4 fallbacks are
  rejected.
- Adds mandatory mutation readback, idempotency, references, rollback, and
  editor/save verification to Visual writes.
- Adds DesignEvidence provenance, CTA destination blocking, native-first plans,
  and explicit approval gates for CSS, JavaScript, or PHP customization.
- Adds versioned site knowledge, draft skill promotion, and 18 P0/P1/P2
  expertise packs with 216 deterministic curriculum contract cases. These do
  not constitute live runtime verification.
- Adds setup diagnostics and direct Streamable HTTP configuration for remote
  clients, with no Node or companion requirement.
- Publishes an exact migration table for all 94 deprecated Elementor add tools.
  They remain in the explicit compatibility surface through the required
  two-release warning window; earliest removal is alpha.67.

## Licensing

- The WordPress plugin and Stonewright Visual are AGPL-3.0-or-later because
  this development line contains attributed Novamira-derived components.
- The Node companion remains MIT.
- Stonewright remains open-source. AGPL adds the obligation to offer the
  corresponding source to users who interact with a modified version over a
  network.

## Security And Control Gates

- `stonewright/php-execute` remains first-class but is bounded, audited, and
  protected by mode, permission, and confirmation gates.
- Elementor, Gutenberg, template, and global-style writes retain backup,
  permission, context, audit, readback, and rollback protection.
- Custom code is proposed separately and never applied without explicit
  approval.
