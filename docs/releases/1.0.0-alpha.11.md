# Stonewright 1.0.0-alpha.11

Release date: 2026-06-11

This alpha tightens visual implementation guidance for design-derived
WordPress pages where the design-tool layer tree is not the same thing as the
visible layout.

## Highlights

- Makes reference screenshots the primary layout authority in
  `stonewright/context-bootstrap` and `stonewright/workflow-preflight`.
- Keeps design-tool data useful for CSS-like details: spacing, colors,
  typography, backgrounds, text, and assets.
- Adds section reference screenshot evidence before visual writes so long
  pages can be compared section-by-section before full-page signoff.
- Updates Elementor and design-to-WordPress skills to build cleaner native
  structures when the visible screenshot conflicts with broken layer nesting.
- Keeps the write control envelope unchanged: writes still require permissions,
  context tokens, backups where required, validation, and production-safe
  confirmation tokens.

## Release Assets

- `stonewright-1.0.0-alpha.11.zip`: installable WordPress plugin package with
  production Composer dependencies.
- `stonewright-companion-1.0.0-alpha.11.tgz`: Node companion package for
  WP-CLI, health checks, and optional MCP proxy.
- `SHA256SUMS.txt`: checksums for release assets.
