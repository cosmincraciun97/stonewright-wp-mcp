# 1.0.0-alpha.75

## Highlights

- **Generic CPT create in Direct mode:** `stonewright-content-create` creates
  items of any registered post type (not only pages/posts).
- **Auto rest_base resolution:** content and taxonomy tools resolve collection
  paths from `/wp/v2/types` and `/wp/v2/taxonomies` so CPTs/taxonomies whose
  `rest_base` differs from the slug work without manual path guessing.
- **session_tools on task-start:** Direct `stonewright-task-start` returns the
  exact enabled tool names for the active profile, plus structured
  content-model capability guidance.
- **Honest capability text:** site-discover and capability tiers state that
  Direct fully edits existing CPT content, taxonomy terms, and ACF field values;
  registering *new* models still needs server-side PHP (WordPress has no REST
  registration endpoint).

## Upgrade notes

- Update the companion for Direct-mode CPT create and rest_base resolution.
  Plugin version is aligned for release packaging; no plugin ability surface
  change in this release.
- After updating an MCP client, restart or reload its MCP servers, call
  `stonewright-task-start`, and use `session_tools` (or honor
  `tools/list_changed` / `re_list_instruction`) so agents see the live tool
  list.
- Prefer `stonewright-content-create` with `type` for any registered post type;
  `stonewright-content-create-page` and `stonewright-content-create-post` remain
  for page/post shortcuts.
