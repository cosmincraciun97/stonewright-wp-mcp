# 1.0.0-alpha.70

## Highlights

- Single source of truth for MCP tool profiles: plugin `tool-profile action=resolve`, companion consumes it.
- Priority ordering so blueprints and brand kits survive client tool caps; optional `STONEWRIGHT_MCP_MAX_TOOLS` trims from the tail.
- Companion emits `tools/list_changed` when profiles change.
- Blueprint `engine=elementor` fails loudly when Elementor is inactive (no silent Gutenberg fallback).
- Gutenberg premium structure: real columns, full-width bands, palette buttons, hero media-text.
- Public surface de-branding with SPDX/reuse ledger preserved; five-release retention policy.

## Upgrade notes

- Prefer Plugin mode for blueprints, Elementor, memory, and audit.
- Set `STONEWRIGHT_MCP_MAX_TOOLS` only when the MCP client has a hard tool cap.
- Pass explicit `engine` when the user names Elementor or Gutenberg.
