# Stonewright 1.0.0-alpha.59

Release date: 2026-06-17

This alpha removes a normal builder-workflow unblock where compact MCP profiles
could hide the Stonewright tools needed for Elementor Loop Grid, Loop Item,
CPT/ACF, and guarded shortcode/query glue work.

## Highlights

- Adds `stonewright/elementor-v3-save-template` to the essential public
  WordPress MCP surface and Elementor compact profiles so agents can create
  Elementor library and Loop Item templates without switching to workarounds.
- Adds guarded sandbox lifecycle tools to the essential, Elementor design, and
  content-model profiles for cases where native Elementor/content tools are not
  enough and shortcode/query glue is required.
- Keeps strict-client `low-tools` under the 30-tool startup cap while preserving
  template save plus sandbox write/activate for Antigravity, Gemini API, and
  similar tool-cap clients.
- Improves the companion WP-CLI runner by inferring the LocalWP PHP binary from
  a site's `conf/php/php.ini` `extension_dir` when possible.
- Adds actionable WP-CLI diagnostics when the selected PHP executable can run
  but WordPress cannot boot because `mysqli`/MySQL extension support is missing.
- Bumps plugin and companion release versions to `1.0.0-alpha.59`.

## Safety Notes

- Sandbox mutations remain guarded by context tokens, permissions,
  `DISALLOW_FILE_MODS`, StaticGuard, audit logging, and production-safe
  confirmation tokens where applicable.
- WP-CLI continues to run only through `execFile` argv tokens and still blocks
  `wp eval`, `wp eval-file`, `wp shell`, `wp package`, `--exec`, and
  `--require`.
- Backup, validation, and Elementor write guards are unchanged.

## Release Assets

- `stonewright-1.0.0-alpha.59.zip`: installable WordPress plugin package with
  production Composer dependencies.
- `stonewright-companion-1.0.0-alpha.59.tgz`: Node companion package for
  WP-CLI, health checks, and optional MCP proxy.
- `SHA256SUMS.txt`: checksums for release assets.
