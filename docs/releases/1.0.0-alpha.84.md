# 1.0.0-alpha.84

## Highlights

- **Native loop wiring.** `stonewright-elementor-wire-loop` plans or adds an
  Elementor Pro Loop Grid/Carousel from compact intent, an existing loop-item
  template, or a validated template spec.
- **Transactional safety.** New templates remain staged until the page write
  passes readback; failures restore the page and remove only the
  transaction-owned template.
- **Live-schema precision.** Template, query, responsive, arrows, and
  pagination intent map only to controls exposed by the installed runtime.
- **Controlled learning.** Schema failures remain non-agentic incidents.
  Matching repairs become active only after two distinct verified tasks or
  explicit operator approval.
- **Serialized writes.** Concurrent writes to the same Elementor post fail
  safely instead of racing and restoring stale state.

## Upgrade notes

- Update the plugin and companion together to `1.0.0-alpha.84`.
- Restart or reload the MCP client after replacing the companion package.
- Start with `stonewright-task-start`, activate the Elementor design profile,
  then call `stonewright-elementor-wire-loop` with `dry_run: true`.
- Existing-template wiring is additive. Creating a template from a spec in
  production-safe mode requires a confirmation token.
