# 1.0.0-alpha.79

## Highlights

- **Reliable session profiles:** task-start and tool-profile activation apply
  the selected profile to the current MCP session while always preserving the
  site-configured surface.
- **Truthful truncation diagnostics:** profile responses report degradation,
  exact omitted tools, and recovery guidance; write-critical Elementor and
  theme tools are ordered first for capped clients.
- **Automatic surface refresh:** the plugin increments `surface_revision` when
  its effective tools change, and the companion re-lists tools and emits
  `notifications/tools/list_changed` when it observes a newer revision.
- **Safer Elementor repair:** touched-node validation preserves unknown
  settings and allows valid mixed V3/V4 documents; transaction writes never
  bypass the integrity gate through raw metadata.
- **Dedicated recovery ability:** `stonewright/elementor-v3-repair-document`
  repairs double-encoded data and duplicate or missing ids with snapshots,
  dry-run support, and idempotent output.
- **Operational clarity:** live registration status, dedicated PHP parse-error
  remediation, and a recovery runbook replace stale or unsupported workarounds.

## Upgrade notes

- Update the plugin and companion together to `1.0.0-alpha.79`.
- Restart or reload the MCP client after replacing the companion package.
- Start the next session with `stonewright-task-start`; if it returns
  `re_list_instruction`, refresh the client tool list before writing.
- For a stale or capped surface, use `stonewright-client-surface-check` and the
  tool-surface recovery runbook instead of direct REST or raw-meta workarounds.
