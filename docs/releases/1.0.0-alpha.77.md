# 1.0.0-alpha.77

## Highlights

- **MCP tool surface sync:** when Setup is Full/essential (or session leaves
  bootstrap), `task-start` always emits `tools_changed` + `re_list_instruction`
  so stdio companions re-register tools such as `php-execute` without REST
  workarounds.
- **Companion** parses ability JSON from `content[].text` when transports omit
  `structuredContent`, then re-registers proxied tools and notifies list change.
- **Bootstrap** cold surface expands to ≤12 tools with runtime escape hatches
  (`php-execute`, confirmation token, content/Elementor reads, theme-file-read).
- **New:** `stonewright-client-surface-check` + `stonewright doctor --client-surface`.
- **New theme abilities:** `theme-file-read` / `theme-file-patch` (allowlist,
  backup, production confirmation; PHP paths need admin gates).
- **Direct Elementor without editor:** `elementor-data-get/update` prefers local
  WP-CLI and falls back to core REST meta on remote when registered (file backup).
  Not a substitute for plugin `elementor-v3-batch-mutate` schema engines.
- **php-execute** advisory `read_only` flag; clearer Elementor write vs read policy.
- **task-start** includes `write_target_url` / active write target labeling.

## Upgrade notes

- Update **plugin and companion together** (same alpha.77).
- After companion upgrade: **restart/reload the MCP client** so tool lists refresh.
- Plugin: call `stonewright-task-start` once; honor `re_list_instruction` if tools
  still look stale, then restart the client if needed.
- Live Elementor edits without the editor: prefer plugin `elementor-v3-batch-mutate`.
  Direct remote uses REST meta only when WordPress exposes `_elementor_data`.
- Do **not** use WP-CLI against remote production unless the companion runs on the
  WordPress host with a real WP root.
