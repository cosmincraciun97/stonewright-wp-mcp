# Stonewright 1.0.0-alpha.66

Alpha.66 repairs the unsafe and low-fidelity paths found after the alpha.65
roadmap release.

## Elementor safety and fidelity

- Workflow preflight now classifies the target document as empty, V3, V4, or
  mixed before recommending a write pipeline.
- Elementor 4 with an empty or unspecified document no longer defaults to V3.
  A caller must explicitly select V3, or use a production-ready V4 adapter.
- V3 writes reject Atomic `e-*` nodes. Mixed documents are blocked instead of
  silently converted.
- Raw `_elementor_data`, page-settings, edit-mode, and version mutation through
  `php-execute` is blocked. Typed abilities remain first-class and mandatory.
- High-level V3 builds now use optimistic tree hashes, strict structural
  readback, snapshot restore on failure, and actionable operation diagnostics.
- The final tree guard rejects missing/duplicate IDs, stripped Unicode escape
  remnants such as `u021b`, common mojibake, and placeholder copy.
- Native flex wrapping and sizing controls survive validation unchanged; batch
  updates that normalize to no change fail instead of claiming success.
- Native container breakpoint settings remain available when Elementor's live
  schema omits its responsive marker, preventing desktop columns from silently
  becoming broken mobile layouts.

## Visual planning

- Figma, screenshot, and image evidence requires a stable reference plus
  SHA-256.
- Visual semantic nodes require measured bounds.
- Native visual planning requires at least two measured viewports, including
  desktop and mobile evidence.
- Strict visual specs require validated DesignEvidence before render.

## Roadmap audit repairs

- Adds the real `stonewright/task-start` compact gateway.
- Keeps 20-tool default and 12-tool strict profiles within budget.
- Keeps unverified expertise packs advisory until live fixture, schema, editor,
  frontend, and readback evidence exists.
- Fixes the companion native-plan allowlist, draft skill reactivation,
  preserved inactive Elementor controls, Gutenberg refs/schema validation, and
  Elementor V4 adapter readback/rollback behavior.
- Adds PHP 8.1-8.5 and Stonewright Visual CI, provenance linting, nested Visual
  skill loading, and reproducible release packaging.

## Verification

Release gates include PHPUnit, PHPStan, PHPCS, security/dependency audits,
provenance lint, token-budget measurement, companion typecheck/tests/build,
Visual typecheck/tests/build, and archive inspection.

The full roadmap remains open where controlled live Elementor and third-party
runtime evidence is still required. Alpha.66 does not pretend otherwise.
