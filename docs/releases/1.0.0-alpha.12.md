# Stonewright 1.0.0-alpha.12

Release date: 2026-06-12

This alpha improves day-to-day agent efficiency and hardens the WordPress admin
controls used to copy setup snippets, reveal secrets, switch panels, and manage
saved memory or skills.

## Highlights

- Adds an OPcache-friendly PHP cache for the Elementor widget manifest so the
  large generated catalog can load without decoding the JSON manifest on every
  request.
- Makes visual guidance conditional: non-visual tasks receive compact visual
  stubs, while Figma, image, prompt, and design-system work still receives the
  full visual quality contract.
- Adds visual page batching guidance: implement one section at a time, or two
  only when simple and coupled, then auto-continue after desktop, tablet, and
  mobile verification passes.
- Fixes admin copy buttons for browsers that reject Clipboard API writes by
  falling back to a temporary textarea copy path and prompt fallback.
- Hardens admin reveal, tab, panel, row, and skill controls so they do not
  accidentally submit surrounding WordPress forms.
- Updates tests for admin JavaScript behaviour and widget manifest loading.

## Release Assets

- `stonewright-1.0.0-alpha.12.zip`: installable WordPress plugin package with
  production Composer dependencies.
- `stonewright-companion-1.0.0-alpha.12.tgz`: Node companion package for
  WP-CLI, health checks, and optional MCP proxy.
