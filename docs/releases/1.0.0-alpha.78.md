# 1.0.0-alpha.78

## Highlights

- **Elementor Integrity Gate (P0):** plugin `ElementorData::write` and Direct
  `elementor-data-update` reject double-encoded JSON, size collapse, and
  widgetType remaps; restore previous document on verified readback rollback.
- **Truthful Elementor errors:** write abilities surface real gate/validator
  codes; context-token errors name canonical `stonewright-task-start`.
- **Memory/learning pipeline:** schema version bumps only after verified
  install; self-heal when version is current but columns are broken; loud
  failures on store/promotion; admin schema-health notice.
- **Retry-storm brake:** 2+ identical ability/tool errors escalate with STOP +
  repair (plugin + Direct).
- **task-start delivery:** companion forwards plugin `initialize.instructions`;
  Direct latch is per-site with 30-minute TTL; non-blocking first-read nudge.
- **Token efficiency:** Direct `elementor-data-get` and V4 atomic-tree reads
  default to capped summary outlines (`TreeSummary`).
- **V4 surgical write:** experimental `stonewright/elementor-v4-update-node`
  (settings-only, dry_run, snapshot, integrity-gated; validates schema
  `$$type` envelopes).
- **Codex hardening:** intentional removals/full rebuilds pass
  `force_destructive`; rollback only claims restored after verify; Direct
  write tools pass site alias (no any-site latch).

## Upgrade notes

- Update **plugin and companion together** (same alpha.78).
- After companion upgrade: **restart/reload the MCP client**.
- Plugin: call `stonewright-task-start` once; honor `re_list_instruction` if
  tools look stale.
- Direct multi-site: task-start must run for the **target site alias** before
  writes; a latch for site A does not unlock site B.
- Large intentional Elementor removals/rebuilds use snapshotted paths with
  `force_destructive`; accidental silent collapse remains blocked on
  surgical patches (e.g. update-element).
