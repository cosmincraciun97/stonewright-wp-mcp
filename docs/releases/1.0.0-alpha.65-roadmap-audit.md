# Alpha.65 Roadmap Audit

Audit date: 2026-07-14  
Repair branch: `codex/context-discipline`

This audit compares the authoritative improvement plan and implementation
prompt with the released alpha.65 tree. It distinguishes source/unit evidence
from controlled WordPress/Elementor runtime evidence.

## Corrections made after alpha.65

- Added the real `stonewright/task-start` ability and made it the essential and
  companion startup path. `context-bootstrap` and `workflow-preflight` remain
  compatibility tools.
- Measured the real compact response: 634 estimated tokens non-visual and 837
  visual after architecture routing. CI exits non-zero if 20/12 tool or 700/1,200 task-start budgets
  regress.
- Fixed the companion essential planner allowlist, disabled-skill reactivation,
  preserved inactive Elementor settings, required unique Elementor node IDs,
  and revision-specific backup metadata.
- Fixed Visual batch refs (`client_id` included), rejected unresolved refs
  before mutation, recursively validated Gutenberg blocks, and enforced block
  attribute types/enums.
- Fixed V4 update/move deltas, undo/redo approval and readback, rollback cache
  cleanup, and clean retry after a failed readback.
- Ported the Novamira Visual skill loader and `use_skill` flow as nested,
  progressive disclosure. Eight adapted AGPL files now have exact source path,
  SHA-256, SPDX, copyright, ledger, and tests.
- Added provenance linting, PHP 8.1-8.5 CI, Visual CI, and reproducible Visual
  packaging/checksums in the tag workflow.
- Stopped self-certifying expertise. The 216 cases are labelled curriculum
  contract audits. Bundled packs without exact live fingerprints are
  `candidate`/advisory-only. Promotion to `verified` requires persisted task,
  fixture, schema hash, editor, frontend, and readback evidence.

## Definition of Done status

Locally demonstrable gates are now explicit and reproducible: tool budgets,
task-start budgets, provenance, PHP quality/security/dependency checks,
companion tests/build, Visual tests/build, and release packaging.

The complete roadmap is **not closed**. Required external evidence remains:

1. controlled Elementor 3.29 Atomic opt-in, Elementor 4.x, and licensed Pro
   editor/frontend/readback matrices with exact fingerprints;
2. live third-party V3 widget/add-on schema cases;
3. evidence-backed expertise scorecards for packs promoted beyond candidate;
4. OAuth PKCE/DCR threat model and E2E client matrix before shipping OAuth;
5. the second released deprecation warning, then actual alias removal no earlier
   than alpha.67;
6. an editor host integration proving `@stonewright/visual` registration in a
   real page, not only its headless package contract.

Synthetic fixtures cannot honestly substitute for those gates.
