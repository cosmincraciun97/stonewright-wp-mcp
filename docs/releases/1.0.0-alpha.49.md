# Stonewright 1.0.0-alpha.49

Released: 2026-06-17

## Highlights

- Makes fresh MCP sessions use the latest companion release so tokenized WP-CLI jobs, stdio startup, and compact handshake guidance are available without local source builds.
- Improves startup behavior for strict MCP clients that need a smaller, clearer initial tool surface.

## Added

- Guarded WP-CLI background jobs through `stonewright-wp-cli-job-start` and `stonewright-wp-cli-job-status`.
- Compact companion MCP handshake instructions for first-call, recovery, low-tools, and WP-CLI routing rules.

## Fixed

- Stale `.env` `PORT` values no longer start the optional HTTP bridge during stdio MCP startup unless `STONEWRIGHT_HTTP_ENABLE=1` or `STONEWRIGHT_HTTP_REQUIRED=1` is set.
- WordPress-side WP-CLI fallback hints now show the explicit HTTP bridge opt-in values.

## Changed

- Companion setup profiles, install snippets, and documentation now point to `1.0.0-alpha.49`.
