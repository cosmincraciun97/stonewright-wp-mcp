# 1.0.0-alpha.80

## Highlights

- **Canonical permanent operating rules:** seven product defaults (responsive
  device tabs, separate verification tab, design section isolation, breakpoint
  isolation, native-first styling, fastest safe interface, verified learning)
  inject in Plugin and Direct modes and stay out of Memory UI.
- **Verified learning receipts:** `stonewright-learning-record` returns
  `stored`, `backend`, `scope`, `memory_id`, `storage_ref`, and `verified:true`
  only after write-then-readback. Accepts canonical `topic`+`correction` and
  legacy Direct `text`.
- **Truthful Stonewright audit coverage:** every ability audit and every
  `stonewright/v1` mutation records exactly one redacted row (`ok` / `error` /
  `blocked`), with exact pagination counts and insert-failure diagnostics.
- **Method ladder:** prefer `typed_api` → `editor_command_bus` → `admin_form` →
  `browser_ui` without skipping safety gates.
- **Breakpoint isolation:** design-derived responsive writes are scoped;
  non-responsive controls return `unsupported_responsive_control` with no write.
- **Design-to-WordPress:** multi-section design nodes require per-section
  evidence and transactions; verification uses a separate browser tab.

## Upgrade notes

- Update the plugin and companion together to `1.0.0-alpha.80`.
- Restart or reload the MCP client after replacing the companion package.
- Start the next session with `stonewright-task-start`; if it returns
  `re_list_instruction`, refresh the client tool list before writing.
- After an explicit “remember this” request, only report success when
  `stonewright-learning-record` returns `verified:true`.
