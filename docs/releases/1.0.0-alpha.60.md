# Stonewright 1.0.0-alpha.60

Release date: 2026-06-17

This alpha hardens local WP-CLI onboarding so agents can tell the difference
between "WP-CLI can launch" and "WordPress can boot through WP-CLI".

## Highlights

- Adds a `php_ini_not_loaded` warning to `stonewright-wp-cli-status` when
  `wp cli info` starts but PHP did not load a `php.ini`, because WordPress
  commands may still fail without mysqli/MySQL extensions.
- Adds `wp_cli_environment` guidance to `stonewright-setup-profile`, covering
  the local/server-side dependencies required for guarded WP-CLI work.
- Updates companion handshake instructions, WordPress agent instructions, and
  admin client prompts so agents stop and report missing local PHP/MySQL,
  WP-CLI, WordPress root, or database prerequisites instead of using shell
  workarounds.
- Clarifies that remote HTTP MCP sites do not require local PHP/MySQL unless
  the companion is expected to run WP-CLI for that site.
- Updates active setup snippets and documentation to `1.0.0-alpha.60`.

## Safety Notes

- WP-CLI still runs only through `execFile` argv tokens and still blocks
  `wp eval`, `wp eval-file`, `wp shell`, `wp package`, `--exec`, and
  `--require`.
- Backup, validation, permission, confirmation-token, and audit gates are
  unchanged.
- This release changes setup diagnostics and agent guidance only; it does not
  add a new write surface.

## Release Assets

- `stonewright-1.0.0-alpha.60.zip`: installable WordPress plugin package with
  production Composer dependencies.
- `stonewright-companion-1.0.0-alpha.60.tgz`: Node companion package for
  WP-CLI, health checks, and optional MCP proxy.
- `SHA256SUMS.txt`: checksums for release assets.
