# 1.0.0-alpha.83

## Highlights

- **Faster next Elementor editor load.** Successful Stonewright writes clear
  generated CSS only for the changed post instead of flushing Elementor's
  global CSS cache.
- **Surgical mixed-document writes.** Existing V3 subtrees remain editable and
  new V3 widgets may be added under an explicit V3 parent. Ambiguous root adds
  and high-level full-document renderers stay blocked.
- **Elementor document health.**
  `stonewright-elementor-document-health` reports serialized size,
  architecture counts, invalid setting paths, and bounded `e-paragraph` ids
  without returning page content.
- **Actionable schema failures.** The first rejected setting now includes its
  path, expected shape, and received type.
- **Verified V4 toggle.** The dashboard checkbox submits explicit checked and
  unchecked values, with bidirectional persistence and feature-gate tests.

## Upgrade notes

- Update the plugin and companion together to `1.0.0-alpha.83`.
- Restart or reload the MCP client after replacing the companion package.
- Start the first task with `stonewright-task-start`, then run
  `stonewright-elementor-document-health` before editing a heavy or mixed
  document.
- Keep mixed-document additions under an explicit V3 parent. A root add remains
  intentionally blocked.
