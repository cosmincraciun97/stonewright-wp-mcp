# 1.0.0-alpha.82

## Highlights

Critical self-improvement and write-safety repair after a live incident where
`php-execute` could append invalid PHP to theme `functions.php` while audit
recorded only snippet return success.

- **`php-execute` never writes code files.** Filesystem mutation APIs are
  permanently blocked (`stonewright_php_code_file_write_blocked`). Use
  `stonewright/theme-file-patch`.
- **Transactional theme-file writes.** Complete-candidate PHP/CSS/JS validation,
  custom-code operator grant, atomic replace, readback, bootstrap smoke, and
  automatic rollback.
- **Custom-code operator grant.** Single-use, short-lived, bound to path +
  candidate `after_sha256`; issued only after an authenticated wp-admin operator
  reviews the bounded diff, native-gap evidence, tests, and rollback plan.
- **Opaque recovery.** Theme backups are non-executable, access-guarded, exposed
  only by opaque references, and restorable through
  `stonewright/theme-backup-restore`.
- **Direct memory scope honesty.** Unknown site aliases fail with
  `site_alias_unresolved` (no silent `_global` write). Plugin-backed Direct uses
  authoritative site memory; pluginless Direct receipts label local-only
  visibility. User-scoped rules remain global across sites, and refreshed
  corrections become newest-first highlights.
- **Effect-oriented audit records.** Plugin SQL and Direct JSONL materialize
  execution / verification / rollback, target, hashes, cause, backend, mode,
  severity, and incident status. REST bodies containing code, skills,
  instructions, or memory are represented only by hashes and byte counts.
- **Breakpoint enforcement in real writes.** Elementor V3 editor mutations
  validate authorized breakpoint scope before writing and roll back when
  non-target breakpoint hashes drift during readback.
- **Lifecycle UI.** Memory separates rules, verified repairs, unresolved
  incidents, audit feedback, and reference entries; legacy feedback migration
  is explicit and preserves history.
- **Safety blocks ≠ learning spam.** Expected blocks do not promote active
  project/user rules; audit feedback stays unresolved until verified repair.
- **Compact startup stays compact.** Target URL, fingerprint, environment, and
  memory backend remain available while task-start stays inside token budgets.
- **Production mode P0 warning** when WordPress environment is production but
  Stonewright mode is not `production-safe`.
- **Canonical rule** `custom_code_operator_grant` (Plugin / Direct / skill parity).

## Upgrade notes

- Update the plugin and companion together to `1.0.0-alpha.82`.
- Restart or reload the MCP client after replacing the companion package.
- On production hosts, switch Stonewright mode to `production-safe` before
  agent writes.
- Theme PHP/CSS/JS apply requires dry_run + custom-code grant; do not retry
  code-file writes through `php-execute`.
- Existing theme backup paths are not accepted by the restore ability; use the
  opaque `backup_ref` returned by a current transactional write.
- Direct learning for an unknown site alias now errors instead of writing
  `_global.jsonl`.
