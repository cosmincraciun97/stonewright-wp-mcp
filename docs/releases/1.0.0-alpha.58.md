# Stonewright 1.0.0-alpha.58

Release date: 2026-06-17

This alpha hardens Stonewright startup guidance so supported agents and IDEs
must use the live Stonewright MCP tool surface for WordPress work.

## Highlights

- Adds shared MCP-use policy text for PHP agent instructions and WordPress admin
  client setup snippets.
- Adds companion setup-profile, handshake, and status guidance that tells
  agents to stop when `stonewright-context-bootstrap` is not visible.
- Explicitly blocks private client config inspection, repository-file
  substitutes, scratch helper scripts such as `query-mcp.js` and
  `run-ability.js`, helper JSON argument files such as `bootstrap-args.json`,
  `cli_command.json`, and `get_structure.json`, ad hoc companion shell launch
  scripts such as `query-local-stonewright.js`, action scripts such as
  `run-loop-mutate.js` and `run-bootstrap-and-mutate.js`, source-code schema
  spelunking, hand-rolled JSON-RPC, REST ability runner shell calls, and shell
  `wp ...` commands as Stonewright MCP workarounds.
- Updates public install, onboarding, companion, client-connection, and
  repository agent guidance so the rule applies across supported clients and
  IDEs, not only Antigravity.
- Bumps plugin and companion release versions to `1.0.0-alpha.58`.

## Safety Notes

- No WordPress write ability changed.
- Backup, confirmation token, permission, validation, and audit gates are
  unchanged.
- The companion still runs guarded WP-CLI through `execFile` argv tokens and
  continues to block arbitrary PHP and shell entry points.

## Release Assets

- `stonewright-1.0.0-alpha.58.zip`: installable WordPress plugin package with
  production Composer dependencies.
- `stonewright-companion-1.0.0-alpha.58.tgz`: Node companion package for
  WP-CLI, health checks, and optional MCP proxy.
- `SHA256SUMS.txt`: checksums for release assets.
