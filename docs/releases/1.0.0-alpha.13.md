# Stonewright 1.0.0-alpha.13

Release date: 2026-06-12

This alpha fixes admin workflows that were confusing or broken during live
site-building QA and makes prompt-enabled skills visible through MCP clients
that support server prompts.

## Highlights

- Exposes prompt-enabled Stonewright skills as MCP prompts from the companion,
  using stable `stonewright-skill-{slug}` names.
- Adds a companion credential store fallback so local per-project Application
  Passwords can be created once through tokenized WP-CLI and reused by future
  agent sessions.
- Fixes Sandbox Library routing so embedded Library tabs, legacy direct
  Widgets/Plugins URLs, and View/Edit/Diff/Delete actions stay on valid admin
  pages.
- Fixes Sandbox active MU plugin detection and keeps action redirects on the
  correct Library subtab.
- Updates the Elementor Icon Box renderer to write `selected_icon`, matching
  Elementor's current widget schema and avoiding blank edit panels.
- Confirms admin copy buttons, skill panels, memory panels, sandbox actions,
  and ability controls in browser QA.

## Release Assets

- `stonewright-1.0.0-alpha.13.zip`: installable WordPress plugin package with
  production Composer dependencies.
- `stonewright-companion-1.0.0-alpha.13.tgz`: Node companion package for
  WP-CLI, health checks, optional MCP proxy, and Stonewright MCP prompts.
- `SHA256SUMS.txt`: checksums for release assets.
